Executive Overview

Onboarding a new customer to your platform is one of the most important — and most error-prone — moments in your relationship with them. Today that process relies on raw spreadsheets passed back and forth over email, manual data entry, and time-consuming back-and-forth between implementation specialists and customers.

Implementation Wizard (Imp Wizard) eliminates that friction. It is a purpose-built web application that guides customers and implementation specialists through every step of the onboarding process — from pulling in their Salesforce account data automatically to generating a clean, validated import file ready for system ingestion.

The core promise

Customers arrive, follow a guided wizard, and leave with a complete, validated implementation package — without a single manual spreadsheet. Specialists spend their time advising, not reformatting data.

The platform serves every segment of the beverage industry: breweries, wineries, cideries, distilleries, kombucha producers, coffee roasters, soda makers, and more — all from a single shared codebase with program-level segmentation to keep each customer's data cleanly separated.

The Problem

Implementation teams today face a predictable set of challenges every time a new customer onboards. These are not edge cases — they happen on every project.

📋

Raw Spreadsheet Hand-offs

Customers receive blank import templates with little guidance. They fill them incorrectly, forget columns, or use wrong formats. Specialists spend hours cleaning the output.

🔁

Repeated Back-and-Forth

Every revision cycle means a new email thread with an attached file. There's no version control, no audit trail, and no visibility into what changed between revisions.

🔍

Manual Catalog Lookup

Specialists manually search a 14,000-item catalog to find matching items for a customer. The process is slow, inconsistent, and entirely dependent on institutional knowledge.

📊

No Visibility or History

Managers and stakeholders have no real-time view of where a project stands. When something goes wrong, there's no audit trail to understand what happened or who changed what.

"By the time we receive a completed template from a customer, we've already emailed it back twice with corrections. The implementation hasn't even started."

Our Solution

Imp Wizard replaces the manual process with a structured, web-based platform that guides both customers and specialists through every step of onboarding. Nothing is left to interpretation — every field has context, every stage has validation, and every action is logged.

Guided 7-Step Wizard

Customers follow a structured flow from business profile to final export. Each step is gated — they can only proceed when the current step is complete and valid.

Salesforce Auto-Fill

The customer's email triggers a live Salesforce lookup. Their account, contact, and opportunity data auto-populates Step 1 — zero re-entry of information that already exists.

14,000-Item Catalog Search

Full-text search across the entire item catalog. Customers and specialists find, select, and stage items in seconds rather than hunting through spreadsheet columns.

Flexible Data Import

Any structured spreadsheet — QuickBooks export, supplier price list, ERP extract — can be uploaded and column-mapped to the right fields. Mappings are saved as named templates for reuse.

Validated XLSX Export

The final output is a clean, validated Excel file formatted exactly for system import. No manual reformatting. No guesswork. Ready to ingest on receipt.

Full Audit Trail

Every action — form submission, item addition, export generation — is logged with timestamp, user, and detail. Every stakeholder can see exactly what happened and when.

The Wizard Experience

The wizard is the heart of the platform. It breaks the onboarding process into seven clearly defined steps, each with its own UI, validation rules, and completion criteria. Customers cannot skip steps or submit incomplete data.

1

Business Profile

Customer enters their company name and contact email. If a matching Salesforce Contact and Opportunity exists, the profile fields auto-populate from CRM data. All fields remain editable.

2

Preferences

Customer selects their producer type (brewery, winery, distillery, etc.), preferred units of measure, and any program-specific configuration fields defined by the implementation team.

3

Catalog Search

Full-text search across 14,000 catalog items classified by producer type. Customers browse, filter, and select items directly into their project's staging area.

4

Flexible Import

Upload any structured file (Excel, CSV, QuickBooks IIF). A column-mapping interface lets users assign their file's columns to target fields. Named import templates can be saved and reused across future projects.

5

Staging Review

All selected and imported items appear in an editable staging table. Users review, correct, reorder, and remove items before committing. Inline editing keeps the workflow fast.

6

Validation

The system runs a full validation pass: required fields, unit conversions, duplicate detection, and format checks. Issues are surfaced inline with clear remediation instructions.

7

Export

A clean, validated .xlsx file is generated and available for download. Export events are pushed back to Salesforce. The implementation record is marked complete.

Customer Portal

Beyond the wizard itself, customers have access to a persistent project portal where they can track progress, complete assigned intake forms, manage their team, and review the full history of their implementation.

Project Dashboard

A single-page view of all projects a customer is associated with, showing status, current step, and any outstanding action items. Customers can see exactly where their implementation stands at any time — no need to email a specialist for a status update.

Intake Forms

Implementation teams assign custom intake forms to projects. Customers complete forms directly in the portal — structured, validated, and tracked. Forms have a controlled lifecycle (Draft → Unlocked → Locked) and changes to published forms are queued for review before taking effect.

Team Management

SuperCustomer users can invite their own colleagues to a project, manage access levels, and remove team members as needed — without involving the implementation team for every change.

History & Audit Trail

Every action taken by the customer or their team is recorded and visible in a project history tab. Customers can see who submitted what, and when — providing transparency and accountability throughout the implementation.

Customers are never left wondering about the status of their project. Everything they need is in one place, accessible 24/7.

Admin & Operations

Implementation specialists and administrators have a full-featured back-office platform for managing projects, configuring the system, and monitoring activity across all customers.

Project Management

Create and manage implementation projects. Assign a CIS specialist. Monitor step progress and project status. Review the full changelog for any project at any time.

Journey Builder

Define reusable implementation journeys — sequences of stages, each containing meetings, resources, and deliverables. Assign a journey to a project to give customers and specialists a structured path to follow. Meeting and resource catalogs provide pre-defined, reusable definitions that can be copied into any journey with a single click.

Audit Log

A searchable, filterable audit log covering every action across the entire platform. Filter by date range, user, role, action type, entity, or project. Export results to CSV for compliance reporting or incident investigation. Timestamps automatically display in each user's configured timezone.

User Management

Invite users, assign roles, manage profiles, and reset passwords — all from the Admin panel. View which projects each user is associated with and what role they hold on each.

Reference Data Management

Full admin control over all reference data: categories, product types, units of measure, implementation types, programs, forms, import templates, and catalog item types. No code changes required to add, rename, or retire any reference value.

Program Segmentation

The platform supports multiple programs — separate tracks for different customer segments, product lines, or internal teams. All data (journeys, forms, templates, projects) is scoped to a program. CIS users can be restricted to only the programs they manage.

Integrations

Salesforce (Bidirectional)

When a customer enters their email in Step 1, the platform performs a live Salesforce Contact lookup. If a matching Contact and linked Opportunity is found, the customer's profile fields auto-populate from CRM data. Status changes and export events are pushed back to Salesforce at key milestones — keeping your CRM current without manual updates.

The integration is optional and configurable. When Salesforce credentials are not provided, the platform runs a no-op service and works fully offline — ideal for development and pilot environments.

QuickBooks (File Upload)

Customers or specialists export data from QuickBooks Desktop (IIF) or QuickBooks Online (CSV) and upload the file directly into the wizard. A column-mapping interface handles the translation — no OAuth registration, no live API connection, and no dependency on QuickBooks availability.

Excel / CSV Import & Export

Any structured spreadsheet from any source can be imported via the flexible column-mapping uploader. The final export is a validated .xlsx file generated by ClosedXML. Both import mappings and export formats are configurable by the implementation team without code changes.

User Roles

The platform has four distinct user roles, each with clearly scoped access and capabilities.

Role Who They Are What They Can Do
Admin Internal platform administrators Full access: all configuration, user management, audit log, all projects, all programs.
CIS Customer Implementation Specialists Manage projects and journeys within their assigned programs. View and export audit log. Invite Customer and SuperCustomer users.
SuperCustomer Primary customer contact (e.g. IT lead, project owner) Full access to their project portal. Complete forms. Invite and manage their team. View project history.
Customer Customer team members Access to assigned projects. Complete assigned forms. View project history and resources.

All users have a self-service profile page where they can update their name, title, organization, and timezone. Email and password changes require verification. All profile changes are audit-logged.

Potential Integrations

The platform is designed with integration in mind. The following integrations are either already built, actively planned, or natural candidates given the platform's architecture.

Salesforce CRM

Bidirectional REST API integration. Pull Contact, Account, and Opportunity data to auto-populate the customer profile at wizard start. Push status changes and export completion events back to keep the CRM current without manual entry.

QuickBooks

File-based integration for both QuickBooks Desktop (IIF) and QuickBooks Online (CSV export). Customers export from QuickBooks and upload directly into the wizard's flexible import step — no OAuth app registration required.

Microsoft Excel / CSV

Native import and export for any structured spreadsheet from any source — supplier price lists, ERP exports, legacy databases. Named column-mapping templates make recurring imports hands-free.

Azure Cloud Platform

The platform is architected for Azure: App Service for the API, Azure Static Web Apps or Blob Storage for the React client, and Azure SQL for the database. All infrastructure-as-code ready for deployment.

Email / Notification Service

Natural integration point for SendGrid, Mailgun, or Azure Communication Services — notify customers when a form is assigned, alert specialists when a form is submitted, or trigger reminders for stalled projects.

ERP / Inventory Systems

The validated XLSX export is structured for direct ingestion by inventory management and ERP platforms. Custom export formats can be configured per program to match any target system's import requirements.

Webhook / Event Hooks

A planned integration layer for outbound webhooks — fire events to any downstream system when key milestones are reached: project created, form submitted, validation passed, export generated.

Single Sign-On (SSO)

The ASP.NET Core Identity foundation supports adding OAuth2 / OpenID Connect providers — Microsoft Entra ID, Google Workspace, or any SAML-compatible identity provider — without changing the application's authorization model.
